CSS学习总结(一)

发表于 2018-05-21 | 分类于 CSS | | 本文总阅读量 次

初次接触CSS的时候，被CSS巨多的知识点所打败，无所适从。尤其是作为一个小白接到一个需求时，甚至都不知道这样的人机交互我该从CSS上入手，还是用JavaScript来做。相信大家初学的时候都深有体会吧。

最近有空将自己学习CSS和项目中写CSS遇到的一些问题总结下来，算是对自己前端知识体系的一个梳理。

基础知识：

1. CSS标准盒模型(W3C盒模型)

外边框边，内填充边，内容边(依次对应Margin Edge，Border Edge，Padding Edge，Content Edge)，content-area，padding-area，border-area，margin-area

2. box-sizing (CSS3属性)

自己在项目中遇到了莫名其妙出现 scrollbar 的情况，找了很久不知道什么原因，最终在 chrome开发者工具 中发现浏览器存在默认设置

body {
    display: block;
    margin: 8px;
}

然后发现div当中子元素总是超出父元素的宽和高导致出现 scrollbar，原来是与 box-sizing 有关，那么什么是box-sizing呢？

box-sizing的作用就是告诉浏览器 width 和 height 用于设置哪一种 box 尺寸。可细分为：

  • content-box = content-area W3C模型默认值
  • padding-box = content-area + padding-area firefox支持
  • border-box = content-area + padding-area + border-area IE模型
  • margin-box = content-area + padding-area + border-area + margin-area

通过查文档了解到chrome默认采用 content-box(W3C模型) 当我们设置了 padding 样式时，就很容易导致子元素的 height 和 width 超出父元素的范围导致出现 scroolbar ，这样的开发体验只能说非常的不友好了。那么这个时候就推荐使用 border-box(IE模型) 了。

但是为了强化浏览器的一致性，在项目中写CSS个人建议采用如下写法规避：

html {
    box-sizing: border-box;
}
*, *::before, *::after {
    box-sizing: inherit;
    margin: 0;
    padding: 0;
    position: relative; //absolute相对于最近的非 `static` 祖先元素定位
}

这样所有的元素除掉了浏览器默认的 margin、padding、box-sizing。而在特定的元素中大家可以使用权重更高的CSS样式选择器去定义。

还有个巨坑那就是设置 div {heiht: 100%;} 的样式问题，我们会发现往往设置没有作用！这是为什么呢，你需要理解浏览器是如何计算高度和宽度的：

Web浏览器在计算有效宽度时会考虑浏览器窗口的打开宽度。如果你不给宽度设定任何缺省值，那浏览器会自动将页面内容平铺填满整个横向宽度。
但是高度的计算方式完全不一样。事实上，浏览器根本就不计算内容的高度，除非内容超出了视窗范围(导致滚动条出现)。或者你给整个页面设置一个绝对高度。否则，浏览器就会简单的让内容往下堆砌，页面的高度根本就无需考虑。
因为页面并没有缺省的高度值，所以，当你让一个元素的高度设定为百分比高度时，无法根据获取父元素的高度，也就无法计算自己的高度。换句话说，父元素的高度只是一个缺省值：height: auto;。当你要求浏览器根据这样一个缺省值来计算百分比高度时，只能得到undefined的结果。也就是一个null值，浏览器不会对这个值有任何的反应。

所以当你写完CSS发现样式很诡异，不是你期望的结果，相信大家心里都会默念一句MMP，都是被浏览器的默认样式给坑了呀！！！

3. 元素的分类

  • 块级元素 display: block、table、flex、grid、list-item
    • 独占一行显示
    • 可以设置尺寸相关属性(width、padding、margin、border)
  • 行内元素 display: inline
    • 可在同一行内排列，默认从左到右排序
    • 不可置换行内元素(img、object、video和embed等) -> 不可设置width、height和垂直方向的margin
    • 可置换行内元素(a和span等) -> 可以设置width、height和垂直方向的margin
    • 水平和垂直方向上的对齐方式，行内元素分别受父元素的text-align属性和自身的vertical-align属性控制，水平方向上默认左对齐
  • 行内块级元素 display: inline-block、inline-table、inline-flex、inline-grid
    • 排列方式与行内元素相同，不独占一行，按从左到右排序
    • 水平和垂直方向上的对齐方式与行内元素同
    • 和块级元素一样，可以设置任何尺寸属性(width默认0)

4. 格式化上下文(Formatting Context)

格式化上下文，它指的是具有某种CSS格式化规则（布局规则）的上下文环境，在这个上下文环境内的所有子元素，都将根据其特定的CSS格式化规则来进行排列。

常见的格式化上下文有BFC（CSS2.1 规范）、IFC（CSS2.1 规范）、 FFC（CSS3规范新增）和GFC（CSS3规范新增）

BFC(block formatting context)

一个独立封闭的渲染区域,在这个区域内的块级元素 ，按从上到下的顺序显示，相邻的块级元素可以使用margin隔离，但在垂直方向上相邻的块级元素会发生margin合并

  • 对应一个独立、封闭的渲染区域，子元素的CSS样式不会影响BFC元素外部
  • 浮动子元素参与BFC父元素的高度计算，也就是BFC元素能够识别浮动元素（解决父元素塌陷）
  • 占据文档流的BFC元素（可使用overflow: auto创建），能够识别浮动的兄弟元素
  • 占据文档流的BFC元素（可使用overflow: auto创建），width为auto时，会占满当前行的剩余宽度

5. 前端布局方案主要有三种：

  • 传统布局方案(借助浮动、定位等手段)
  • flex布局方案
  • grid布局方案

传统布局方案：基础源于CSS2，浏览器兼容性最好，但实现复杂度高，学习成本大
flex布局方案：学习成本低，简化布局复杂度，>=IE10
grid布局方案：微软提出

定位(position)

工作后第一次当面试官的体验，深感前端圈面试css的position能刷掉一半人，其中甚至不乏很多3、4年工作经验的人这个梗不是空穴来风的。

position属性的取值有:

  • static（默认值）静态定位：无法设置top、right、bottom和left这四个偏移属性
  • relative 相对定位：相对元素在文档流原来的位置
  • absolute 绝对定位：离元素最近的一个非static（包含relative、absolute和fixed）定位祖先元素
  • fixed 固定定位：相对当前的视窗（viewport）

6. CSS权重

说了这么多基础知识，那么最重要的需要搞懂的是CSS样式的优先级问题，做项目的时候会遇到不同地方对元素定义不同的CSS样式的问题，那么搞懂浏览器渲染CSS的优先级就很重要了

同一个HTML元素被定义了多个CSS样式(通过class选择器、id选择器、行内样式等定义)时，css样式的优先渲染依赖CSS权重判断

权重的五个等级：

  • !important
  • 行内样式
  • id选择器，权重100
  • class选择器、属性选择器和伪类选择器，权重10
  • HTML标签选择器和伪元素选择器，权重1

权重之间的关系：

  • !important > 行内样式 > id选择器 > 类选择器 | 属性选择器 | 伪类选择器 > 元素选择器
  • 权重大的选择器覆盖权重小的选择器css样式规则
  • 同等权重大小的选择器，以最后规定的选择器为准
